## 🔍 Detection Rules

### Sigma Rule: Brute Force Detection

**File**: `detections/sigma_bruteforce.yml`

**Detection Logic**:
- Monitors Event IDs 4625 (failed logon) and 4624 (successful logon)
- Triggers on 5+ failed attempts from same source IP within 5 minutes
- Followed by successful authentication

**MITRE ATT&CK**: T1110.001 (Brute Force: Password Guessing)

### Sigma Rule: Suspicious PowerShell

**File**: `detections/sigma_suspicious_powershell.yml`

**Detection Logic**:
- Encoded commands (`-EncodedCommand`, `-enc`)
- Download cradles (`DownloadString`, `WebClient`, `IWR`)
- Execution policy bypass (`-ExecutionPolicy Bypass`)
- Hidden window execution (`-WindowStyle Hidden`)
- Invoke-Expression (`IEX`)

**MITRE ATT&CK**: T1059.001 (PowerShell), T1027 (Obfuscation), T1105 (Ingress Tool Transfer)

## 📈 Investigation Script Features

### Main Script (`src/investigate.py`)

- **Input**: CSV log file with Windows Security and PowerShell events
- **Output**: 4 CSV files with analysis results
- **CLI Interface**: Argparse-based with help documentation

### Detection Functions (`src/utils.py`)

1. **`detect_brute_force()`**
   - Sliding window analysis (5-minute windows)
   - Groups events by host and source IP
   - Configurable threshold (default: 5 failed attempts)

2. **`detect_suspicious_powershell()`**
   - Pattern matching for malicious indicators
   - Risk scoring system (0-100)
   - Base64 decoding for encoded commands

3. **`calculate_severity()`**
   - Brute Force: Low (<5), Medium (5-9), High (10+)
   - PowerShell: Low (<30), Medium (30-59), High (60+)

4. **`generate_timeline()`**
   - Chronological event ordering
   - Detection type tagging
   - Severity assignment

## 🔧 Customization

### Adjusting Detection Thresholds

Edit `src/investigate.py` to modify detection parameters:

```python
# Brute force threshold (default: 5 failed attempts in 5 minutes)
brute_force_hits = detect_brute_force(df, threshold=5, time_window_minutes=5)
```

### Adding New Indicators

Extend `src/utils.py` to add PowerShell indicators:

```python
# In detect_suspicious_powershell() function
new_indicators = ['malicious_pattern', 'suspicious_keyword']
if any(indicator in command_line for indicator in new_indicators):
    indicators.append('New Indicator Type')
    risk_score += 25
```

## 🛡️ Real-World Application

### Deploying Sigma Rules

These Sigma rules can be converted to various SIEM formats:

```bash
# Convert to Splunk
sigmac -t splunk detections/sigma_bruteforce.yml

# Convert to Elastic
sigmac -t es-qs detections/sigma_suspicious_powershell.yml

# Convert to QRadar
sigmac -t qradar detections/sigma_bruteforce.yml
```

### Integration with SIEM

The Python script can be adapted to:
- Query logs directly from SIEM APIs (Splunk, Elastic, QRadar)
- Run as scheduled job for continuous monitoring
- Send alerts via email/Slack/PagerDuty
